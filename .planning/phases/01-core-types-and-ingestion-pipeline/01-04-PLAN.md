**Types:**

```rust
pub struct SyncFileResult {
    pub file_path: String,
    pub records_synced: usize,
    pub records_failed: usize,
    pub warnings: Vec<ParseWarning>,
    pub overflow_fields_logged: usize,
    pub skipped: bool,  // true if no new data
}

pub struct SyncAllResult {
    pub files_discovered: usize,
    pub files_synced: usize,
    pub files_skipped: usize,  // no new data
    pub files_errored: usize,
    pub total_records: usize,
    pub total_warnings: usize,
    pub total_overflow_fields: usize,
}
```

**sync_file function:**

`pub async fn sync_file(conn: &tokio_rusqlite::Connection, path: &Path, session_id: &str) -> Result<SyncFileResult>`

Implementation:
1. Inside `conn.call()`, query sync_metadata for last_byte_offset: `SELECT last_byte_offset FROM sync_metadata WHERE file_path = ?1` — default to 0 if not found
2. Get file size. If file_size <= last_offset, return SyncFileResult with skipped=true [SYNC-01]
3. Call `parse_jsonl(path, last_offset)` to get ParseResult [uses CORE-06, CORE-07]
4. If no records parsed and no warnings, still update offset (file may have only empty lines)
5. Process records in batches of 1000 [SYNC-03]:
   - Begin unchecked_transaction
   - For each record in batch: call decompose_record(record, session_id, &tx)
   - Call log_overflow for each record's overflow
   - After batch: update sync_metadata atomically in same transaction [SYNC-04]:
     ```sql
     INSERT INTO sync_metadata (file_path, last_byte_offset, record_count, last_synced_at)
     VALUES (?1, ?2, ?3, datetime('now'))
     ON CONFLICT(file_path) DO UPDATE SET
       last_byte_offset = ?2,
       record_count = record_count + ?3,
       last_synced_at = datetime('now')
     ```
     where ?2 is the new_offset from ParseResult and ?3 is the batch record count
   - Commit transaction
6. Log summary at INFO level: "Synced {path}: {records} records, {warnings} warnings"
7. Return SyncFileResult with totals

**sync_all function:**

`pub async fn sync_all(conn: &tokio_rusqlite::Connection, projects_dir: &Path) -> Result<SyncAllResult>`

Implementation [SYNC-02]:
1. Use walkdir to recursively discover all .jsonl files under projects_dir
2. For each .jsonl file:
   - Extract session_id from the filename (stem without extension): the filename is a UUID like `eb3ca04b-0383-4955-8606-7c5c9cabe3d7.jsonl`
   - Call sync_file(conn, &file_path, &session_id)
   - Accumulate results into SyncAllResult
   - On error: log warning with file path and error, increment files_errored, continue to next file (never halt bulk import)
3. Log summary at INFO level: "Sync complete: {files_synced}/{files_discovered} files, {total_records} records"
4. Return SyncAllResult

**Session ID extraction:**
- For main session files: `~/.claude/projects/{project-path}/{session-uuid}.jsonl` — session_id = filename stem
- For subagent files: `~/.claude/projects/{project-path}/{session-uuid}/subagents/agent-{agent-id}.jsonl` — session_id = the parent directory's session UUID (two levels up from "subagents"), NOT the filename. Actually, per the research, the filename for subagent files is `agent-{agent-id}.jsonl`, so the session_id should be extracted from the parent path. Create a helper: `fn extract_session_id(path: &Path) -> Option<String>` that handles both cases.

Update `crates/store/src/lib.rs` to add `pub mod sync;`.

**CLI structure with clap derive:**

```rust
#[derive(Parser)]
#[command(name = "claude-history", about = "Claude Code session history API")]
struct Cli {
    #[command(subcommand)]
    command: Commands,
}

#[derive(Subcommand)]
enum Commands {
    /// Sync JSONL session files into the database
    Sync {
        /// Path to Claude Code projects directory
        /// Defaults to ~/.claude/projects/
        #[arg(long)]
        projects_dir: Option<PathBuf>,

        /// Path to the database file
        /// Defaults to $CLAUDE_HISTORY_DB_PATH or ~/.claude/.claude-history.db
        #[arg(long)]
        db_path: Option<PathBuf>,
    },
}
```

**sync command handler:**

1. Resolve db_path: CLI arg > env var CLAUDE_HISTORY_DB_PATH > default ~/.claude/.claude-history.db
2. Resolve projects_dir: CLI arg > default ~/.claude/projects/
3. Validate projects_dir exists (error with helpful message if not)
4. Create parent directories for db_path if needed
5. Call `init_db(&db_path).await` to open and migrate
6. Call `sync_all(&conn, &projects_dir).await`
7. Print human-readable summary to stdout:
   ```
   Sync complete:
     Files discovered: {N}
     Files synced:     {N}
     Files skipped:    {N} (no new data)
     Files errored:    {N}
     Records ingested: {N}
     Warnings:         {N}
     Drift fields:     {N}
   ```
8. Exit with code 0 on success, 1 if any files errored

**Logging setup:**
Initialize tracing-subscriber with env-filter at the top of main. Default log level INFO. Set RUST_LOG=debug for verbose output. Logs go to stderr (via tracing-subscriber default), summary output goes to stdout.
