  <action>
    Implement the async handler functions for each new subcommand, following the existing run_sync pattern. Each handler:
    1. Resolves db_path using the existing resolve_db_path function (add --db-path arg to each subcommand OR extract db_path resolution to a shared location -- the simpler approach is a global --db-path on the Cli struct).
    2. Opens the database via claude_history_store::db::init_db.
    3. Calls the appropriate store query function inside conn.call().
    4. Formats output using output.rs helpers based on --json flag.
    5. Returns ExitCode::SUCCESS or ExitCode::FAILURE.

    **Important architectural choice for --db-path:** Move `db_path: Option<PathBuf>` from the Sync variant to the top-level Cli struct as a global argument. This avoids repeating it on every subcommand. The Sync variant keeps `projects_dir` as its unique arg. Update run_sync to accept the db_path from the Cli-level field.

    **run_search(conn, query, limit, json) -> ExitCode** [CLI-05]:
    - conn.call(move |conn| fts::search_messages(conn, &query, limit, 0))
    - If json: output::print_json(&results)
    - If not json: output::print_search_results(&results)
    - Print result count summary to stderr

    **run_sessions(conn, project, after, before, limit, json) -> ExitCode** [CLI-04]:
    - conn.call(move |conn| query::list_sessions(conn, project.as_deref(), after.as_deref(), before.as_deref(), limit))
    - If json: output::print_json(&results)
    - If not json: output::print_sessions_table(&results)

    **run_query(conn, session_id, message_type, model, tool, after, before, limit) -> ExitCode** [CLI-03]:
    - conn.call(move |conn| query::query_messages(conn, ...all filters...))
    - Always output JSON (this subcommand is for machine consumption per spec: "outputs matching messages as JSON to stdout")
    - output::print_json(&results)

    **run_stats(conn, session_id, json) -> ExitCode** [CLI-06]:
    - Three queries: token_stats_by_model (or by_session if session_id provided), tool_frequency, model_breakdown
    - If json: combine into a JSON object with keys "token_usage", "tool_frequency", "model_breakdown"
    - If not json: output::print_stats(&token_stats, &tool_stats, &model_stats)

    Error handling: wrap conn.call errors with eprintln to stderr, return ExitCode::FAILURE. Do NOT use unwrap/expect — follow the existing error handling pattern in run_sync.

    The conn.call closure captures moved values. Clone Strings from the clap args before moving into the closure. The closure signature is `|conn: &Connection| -> Result<T, rusqlite::Error>`.
