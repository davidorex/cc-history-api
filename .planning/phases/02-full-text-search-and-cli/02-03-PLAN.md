    **export.rs** [CLI-07] — Create module for session export in three formats. All export functions write directly to a `Write` impl (typically stdout) to support streaming and avoid OOM on large sessions (Research Pitfall 5).

    1. `export_json(conn: &Connection, session_id: &str, writer: &mut impl std::io::Write) -> Result<(), Box<dyn std::error::Error>>`:
       - Load session metadata from sessions table.
       - Load messages in batches (100 at a time) via query::session_messages_for_export.
       - Build a JSON structure: `{ "session_id": "...", "project_path": "...", "messages": [...] }`.
       - For practical simplicity, collect all messages into a Vec then serde_json::to_writer_pretty. If the session is very large (>5000 messages), the batch loading prevents OOM at the query level; the serialization step may still hold the full Vec. This is an acceptable trade-off for Phase 2. A streaming JSON serializer could be a Phase 3+ optimization.
       - Each message includes: uuid, type, timestamp, model, stop_reason, content_blocks (array), token_usage (if present).

    2. `export_markdown(conn: &Connection, session_id: &str, writer: &mut impl std::io::Write) -> Result<(), Box<dyn std::error::Error>>`:
       - Header: `# Session: {session_id}\n**Project:** {project_path}\n**Date:** {first_seen_at}\n`
       - For each message (loaded in batches):
         - `---\n## {Type} ({timestamp})\n\n`
         - For text/thinking blocks: print text_content directly.
         - For tool_use blocks: `### Tool Use: {tool_name}\n**Input:** \`{truncated_input}\`\n` (truncate input to 200 chars).
         - For tool_result blocks: print text_content truncated to 500 chars with a note if truncated.
       - Write each message as it's processed (streaming to writer).

    3. `export_csv(conn: &Connection, session_id: &str, writer: &mut impl std::io::Write) -> Result<(), Box<dyn std::error::Error>>`:
       - Use `csv::Writer::from_writer(writer)`.
       - Header row: uuid, session_id, type, timestamp, model, content_preview, input_tokens, output_tokens.
       - For each message: content_preview is the first text block's content truncated to 500 chars.
       - Token usage from the token_usage fields if present, else empty.
       - The csv crate handles quoting and escaping (Research: Don't Hand-Roll CSV).

    **main.rs** — Add Export variant to Commands enum:
    ```rust
    Export {
        /// Session ID to export
        session_id: String,
        /// Output format: json, markdown, csv
        #[arg(long, default_value = "json")]
        format: String,
    },
    ```

    Add `mod export;` declaration.

    Implement `run_export(conn, session_id, format) -> ExitCode`:
    - Validate format is one of "json", "markdown", "csv" — print error and return FAILURE if not.
    - Lock stdout for efficiency: `let stdout = std::io::stdout(); let mut handle = stdout.lock();`
    - Match on format and call the appropriate export function.
    - On error, eprintln to stderr and return FAILURE.
    - conn.call closure loads data, but writing happens outside the closure. The export functions need the Connection directly (synchronous), so they run inside conn.call. This means the export writes to a buffer inside the closure, then the buffer is written to stdout outside. Alternative: pass a Vec<u8> as the writer, return it from the closure, then write to stdout. This avoids blocking the DB connection thread with I/O.

    Wire the Export match arm in the main match statement.

    ```rust
    /// Show Claude Code version history detected from ingested data
    VersionCheck {
        /// Output as JSON

        json: bool,
    },
    /// Show schema drift events (unknown fields captured during ingestion)
    SchemaDrift {
        /// Filter by record type
        #[arg(long)]
        record_type: Option<String>,
        /// Maximum entries to show
        #[arg(long, default_value = "50")]
        limit: usize,
        /// Output as JSON
        #[arg(long)]
        json: bool,
    },
    ```

    **run_version_check(conn, json) -> ExitCode** [CLI-08]:
    - conn.call(move |conn| query::version_history(conn))
    - If json: output::print_json(&results)
    - If not json: print a table with VERSION, FIRST_SEEN, LAST_SEEN columns. Use column-aligned formatting.
    - If no versions found, print "No version data found. Run 'claude-history sync' first." to stderr.

    **run_schema_drift(conn, record_type, limit, json) -> ExitCode** [CLI-09]:
    - conn.call(move |conn| query::schema_drift_list(conn))
    - Apply record_type filter in Rust if provided (filter the Vec after retrieval). This keeps the query.rs function simple. If performance matters later, the filter can move to SQL.
    - Apply limit truncation.
    - If json: output::print_json(&results)
    - If not json: print a table with FIELD, RECORD_TYPE, VERSION, FIRST_SEEN, SAMPLE_VALUE columns. Truncate sample_value to 50 chars for display.
    - If no drift entries, print "No schema drift detected." to stderr.

    Wire both match arms in the main match statement.
