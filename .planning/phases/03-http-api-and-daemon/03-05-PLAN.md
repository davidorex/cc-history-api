<task type="auto">
  <name>Task 2: Implement DaemonClient with HTTP-over-UDS and ConnectionMode enum</name>
  <files>crates/server/src/daemon_client.rs</files>
  <action>Create crates/server/src/daemon_client.rs. Define ConnectionMode enum: pub enum ConnectionMode { Daemon(DaemonClient), Direct { conn: tokio_rusqlite::Connection, db_path: std::path::PathBuf } }. Define pub struct DaemonClient { socket_path: PathBuf }. Implement a private async method request(&self, method: &str, path: &str, body: Option<&[u8]>) -> Result<Vec<u8>, DaemonError> that: (1) Opens a tokio::net::UnixStream to self.socket_path. (2) Uses hyper_util::rt::TokioIo to wrap the UnixStream. (3) Sends an HTTP/1.1 request with the given method, path, Host header set to 'localhost', and optional body with Content-Type: application/json. Use hyper::client::conn::http1::handshake() to establish the connection, then send_request(). (4) Collects the response body using http_body_util::BodyExt::collect(). (5) If status is not 2xx, return DaemonError::Api { status, body }. (6) Returns the collected body bytes. Define DaemonError enum: Connection(std::io::Error), Hyper(hyper::Error), Api { status: u16, body: String }, Json(serde_json::Error). Implement Display and std::error::Error for DaemonError. Add 10 convenience methods mapping to CLI subcommand needs: health, sessions, search, query_messages, stats_tokens, stats_tools, stats_models, export_session, version_history, schema_drift. Add pub fn resolve_socket_path(cli_arg: Option<&std::path::Path>) -> PathBuf: checks cli_arg, then CLAUDE_HISTORY_SOCKET env, then /tmp/claude-history.sock default. Add pub async fn detect_connection_mode(socket_path: &Path, db_path_arg: Option<PathBuf>) -> ConnectionMode: if socket exists AND a test GET /v1/health succeeds, return Daemon(DaemonClient); otherwise fall back to Direct with open_db. The health check verifies the daemon is actually responsive, not just that a stale socket file exists.</action>
