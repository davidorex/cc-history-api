   - notify = { version = "8.2", features = ["macos_fsevent"] }
   - tokio-stream = { version = "0.1", features = ["sync"] }
   - futures-util = "0.3"

2. In crates/server/Cargo.toml, add to [dependencies]:
   - tokio-stream = { workspace = true }
   - futures-util = { workspace = true }
   (notify will be added in Plan 02 when the watcher module is created)

3. Create crates/server/src/events.rs with:
   a. SseEvent enum with 4 variants matching the spec requirement names exactly:
      - RecordAdded { session_id: String, records_synced: usize, file_path: String } -> SSE event name "record:added"
      - SessionStarted { session_id: String } -> SSE event name "session:started"
      - SchemaDrift { new_fields: usize, session_id: String } -> SSE event name "schema:drift"
      - VersionChanged { old_version: Option<String>, new_version: String, session_id: String } -> SSE event name "version:changed"
      Derive Clone, Debug, serde::Serialize, serde::Deserialize on SseEvent.
      Do NOT use serde(tag/content) for the enum — the SSE event name is set via axum's Event::event() method, and the data payload is the struct fields serialized as JSON. Use a method like `fn event_type(&self) -> &'static str` that returns the SSE event name string, and a `fn to_json_data(&self) -> serde_json::Value` that serializes the variant's data fields.

   b. events_handler async function:
      - Signature: `pub async fn events_handler(State(state): State<SharedState>) -> Sse<impl Stream<Item = Result<Event, Infallible>>>`
      - Subscribe to state.event_tx via `.subscribe()`
      - Wrap the broadcast::Receiver in `tokio_stream::wrappers::BroadcastStream`
      - Use `.filter_map()` to silently drop `BroadcastStreamRecvError::Lagged` errors (slow consumers lose events gracefully)
      - Map each SseEvent to `axum::response::sse::Event::default().event(sse_event.event_type()).json_data(sse_event.to_json_data())` — handle json_data errors with a fallback `Event::default().data("serialization error")`
      - Return `Sse::new(stream).keep_alive(KeepAlive::default())`
      - Required imports: axum::response::sse::{Event, KeepAlive, Sse}, axum::extract::State, tokio_stream::wrappers::BroadcastStream, tokio_stream::StreamExt (for filter_map/map), std::convert::Infallible, futures_util::stream::Stream

4. Add `pub mod events;` to crates/server/src/lib.rs (the server crate's lib.rs, if it exists — otherwise add to main.rs module declarations).

Note: The server crate uses `tokio_rusqlite::rusqlite` re-export pattern for rusqlite access (decision [02-03]). The events module does not need direct rusqlite access.
