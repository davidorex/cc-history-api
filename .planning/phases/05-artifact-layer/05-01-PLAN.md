<task type="auto">
  <name>Task 1: Create migration 003_artifacts.sql with files, file_operations, git_operations tables and FTS5 index</name>
  <files>crates/store/migrations/003_artifacts.sql, crates/store/src/schema.rs</files>
  <action>
Create crates/store/migrations/003_artifacts.sql with:

1. **files table** [ART-01]: One row per unique file path per session.
   - id INTEGER PRIMARY KEY AUTOINCREMENT
   - session_id TEXT NOT NULL REFERENCES sessions(session_id)
   - file_path TEXT NOT NULL
   - first_seen TEXT NOT NULL
   - last_modified TEXT NOT NULL
   - operation_count INTEGER NOT NULL DEFAULT 0
   - UNIQUE(session_id, file_path)

2. **file_operations table** [ART-02]: Every file touch operation.
   - id INTEGER PRIMARY KEY AUTOINCREMENT
   - session_id TEXT NOT NULL REFERENCES sessions(session_id)
   - file_path TEXT NOT NULL
   - operation_type TEXT NOT NULL (write, edit, read, bash_cp, bash_mv, bash_rm, bash_mkdir, bash_touch)
   - content TEXT (full content for write, new_string for edit, NULL for read)
   - old_content TEXT (old_string for edit, NULL for write/read)
   - command TEXT (Bash command for bash_* operations)
   - tool_use_id TEXT
   - message_uuid TEXT REFERENCES messages(uuid)
   - timestamp TEXT NOT NULL
   - UNIQUE(tool_use_id) -- each tool_use produces at most one file operation

3. **git_operations table** [ART-03]: Extracted from Bash git commands.
   - id INTEGER PRIMARY KEY AUTOINCREMENT
   - session_id TEXT NOT NULL REFERENCES sessions(session_id)
   - operation_type TEXT NOT NULL (commit, push, checkout, branch, merge, rebase, stash, pull, status, log, diff, add)
   - command TEXT NOT NULL (full Bash command)
   - commit_message TEXT (extracted for commit operations)
   - branch TEXT (extracted where detectable)
   - tool_use_id TEXT
   - message_uuid TEXT REFERENCES messages(uuid)
   - timestamp TEXT NOT NULL
   - UNIQUE(tool_use_id, operation_type) -- one tool_use can produce multiple git ops (add && commit)

4. **Indexes** following PAT-032 naming convention idx_{table}_{column}:
   - idx_files_session_id ON files(session_id)
   - idx_files_file_path ON files(file_path)
   - idx_file_operations_session_id ON file_operations(session_id)
   - idx_file_operations_file_path ON file_operations(file_path)
   - idx_file_operations_timestamp ON file_operations(timestamp)
   - idx_file_operations_tool_use_id ON file_operations(tool_use_id)
   - idx_git_operations_session_id ON git_operations(session_id)
   - idx_git_operations_operation_type ON git_operations(operation_type)
   - idx_git_operations_timestamp ON git_operations(timestamp)

5. **FTS5 virtual table** [FTS-02]: External-content FTS5 over file_operations for content search.
   - CREATE VIRTUAL TABLE fts_file_operations USING fts5(content_col, old_content_col, command_col, content='file_operations', content_rowid='id', tokenize='unicode61')
   - This follows the same external-content pattern as 002_fts5.sql (fts_message_content)

Then add the migration to schema.rs MIGRATIONS array:
```rust
("003", include_str!("../migrations/003_artifacts.sql")),
```
This follows PAT-010 (embedded migrations via include_str!).
