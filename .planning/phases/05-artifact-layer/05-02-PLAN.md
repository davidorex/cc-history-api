Create crates/store/src/artifacts.rs with module-level doc comments explaining the artifact decomposition pipeline.

**Public entry point:**
```rust
pub fn decompose_artifacts(
    record: &JSONLRecord,
    session_id: &str,
    tx: &Transaction,
) -> Result<usize, DecomposeError>
```
Dispatches to decompose_assistant_artifacts for Assistant records and decompose_user_artifacts for User records (User artifacts may include Bash tool results; however, the primary file/git extraction happens from the assistant's tool_use blocks). Return 0 for other record types.

**decompose_assistant_artifacts:** Iterate over `r.message.content` blocks. For each `ContentBlock::ToolUse { id, name, input, .. }`, dispatch on `name.as_str()`:
- "Write" -> extract_write_operation [ART-05]
- "Edit" -> extract_edit_operation [ART-06]
- "Read" -> extract_read_operation [ART-07]
- "Bash" -> extract_bash_operations [ART-08, ART-09]
- Other tool names -> skip silently

**extract_write_operation [ART-05]:**
Parse input JSON: file_path (string), content (string). Both required.
1. Call upsert_file(session_id, file_path, timestamp, tx)
2. INSERT OR IGNORE INTO file_operations (session_id, file_path, operation_type='write', content=content, old_content=NULL, command=NULL, tool_use_id=id, message_uuid=uuid, timestamp=timestamp)
3. UPDATE files SET operation_count = operation_count + 1, last_modified = MAX(last_modified, timestamp) WHERE session_id=? AND file_path=?

**extract_edit_operation [ART-06]:**
Parse input JSON: file_path (string), old_string (string), new_string (string). All required. Also extract replace_all (bool, default false) -- store it in the content field as a JSON object `{"new_string": "...", "replace_all": bool}` or just store new_string in content and old_string in old_content. Research recommends storing new_string in content and old_string in old_content for reconstruction simplicity.
1. Call upsert_file
2. INSERT OR IGNORE INTO file_operations (session_id, file_path, operation_type='edit', content=new_string, old_content=old_string, command=NULL, tool_use_id=id, message_uuid=uuid, timestamp=timestamp)
3. UPDATE files SET operation_count = operation_count + 1, last_modified = MAX(last_modified, timestamp)

**extract_read_operation [ART-07]:**
Parse input JSON: file_path (string). Required.
1. Call upsert_file
2. INSERT OR IGNORE INTO file_operations (session_id, file_path, operation_type='read', content=NULL, old_content=NULL, command=NULL, tool_use_id=id, message_uuid=uuid, timestamp=timestamp)
3. UPDATE files SET last_modified = MAX(last_modified, timestamp) -- no operation_count increment for reads

**extract_bash_operations [ART-08, ART-09]:**
Parse input JSON: command (string). Required. description (string) is optional, can be ignored.
1. Check for git commands using regex. Use `regex::Regex` (compile once with `std::sync::OnceLock` or `lazy_static` pattern -- prefer OnceLock as it is std). Key patterns:
   - Git operation detector: `(?:^|&&\s*|;\s*)git\s+(\w+)` to find git subcommands
   - HEREDOC commit message: `git\s+commit\s+.*?-m\s+"\$\(cat\s+<<'?EOF'?\n([\s\S]*?)\nEOF` (multiline)
   - Inline commit message: `git\s+commit\s+.*?-m\s+"([^"]+)"`
   - Branch extraction: `git\s+(?:checkout\s+-b|push\s+\w+)\s+(\S+)`
2. For each git subcommand found, INSERT OR IGNORE INTO git_operations with appropriate operation_type, commit_message, branch
3. Check for file-touching shell commands: cp, mv, rm, mkdir, touch. Extract file paths where possible.
   - Pattern: `(?:^|&&\s*|;\s*)(cp|mv|rm|mkdir|touch)\s+(.+?)(?:\s*(?:&&|;|$))`
   - INSERT OR IGNORE INTO file_operations with operation_type = "bash_{cmd}", command = full command
   - Call upsert_file for each extracted file path

**upsert_file helper:**
```rust
fn upsert_file(session_id: &str, file_path: &str, timestamp: &str, tx: &Transaction) -> Result<(), DecomposeError> {
    tx.execute(
        "INSERT INTO files (session_id, file_path, first_seen, last_modified, operation_count)
         VALUES (?1, ?2, ?3, ?3, 0)
         ON CONFLICT(session_id, file_path) DO NOTHING",
        rusqlite::params![session_id, file_path, timestamp],
    )?;
    Ok(())
}
```

Use INSERT OR IGNORE for all inserts (PAT-012 idempotency pattern). Use std::sync::OnceLock for regex compilation (avoids external dependency on lazy_static).
