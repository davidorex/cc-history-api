<objective>
Create the artifact query layer: functions for listing files, querying operations, reconstructing file content via edit replay, generating unified diffs, and building session artifact timelines.

Purpose: These query functions are the read-side counterpart to the artifact decomposer. They provide the data retrieval that the API handlers (Plan 06) and CLI subcommands (Plan 07) will call. The reconstruction algorithm is the most technically interesting piece -- it replays Write and Edit operations in timestamp order to reproduce file content at any point in a session.

Output: New artifact_queries.rs module with all query functions, updated lib.rs.

**Query functions:**

1. `list_files(conn, session_id: Option<&str>, path_contains: Option<&str>, limit: usize) -> Result<Vec<FileEntry>>` [for API-17, CLI-10]
   - SELECT from files with optional session_id and file_path LIKE '%..%' filters
   - ORDER BY last_modified DESC, LIMIT

2. `get_file(conn, file_id: i64) -> Result<Option<FileEntry>>` [for API-18]
   - SELECT from files WHERE id = ?

3. `query_file_operations(conn, file_path: &str, session_id: Option<&str>, limit: usize) -> Result<Vec<FileOperation>>` [for API-18, CLI-11]
   - SELECT from file_operations WHERE file_path = ? with optional session_id filter
   - ORDER BY timestamp ASC, LIMIT

4. `query_file_operations_ordered(conn, file_path: &str, session_id: &str, cutoff_timestamp: Option<&str>) -> Result<Vec<FileOperation>>` [internal, for reconstruction]
   - SELECT from file_operations WHERE file_path = ? AND session_id = ?
   - Optional: AND timestamp <= cutoff_timestamp
   - ORDER BY timestamp ASC (no limit, needs all ops for replay)

5. `reconstruct_file_content(conn, file_path: &str, session_id: &str, at_message_uuid: Option<&str>) -> Result<Option<String>>` [ART-10, for API-19, CLI-12]
   - If at_message_uuid is provided, look up its timestamp from messages table
   - Call query_file_operations_ordered with cutoff
   - Replay: start with content = None
     - "write" => content = Some(op.content.clone())
     - "edit" => if content exists and old_content/content present, apply: content = content.replace(old_content, new_content)
     - Other types (read, bash_*) => no mutation
   - Return final content

6. `generate_file_diff(conn, file_path: &str, session_id: &str) -> Result<String>` [ART-11, for API-20]
   - Call query_file_operations_ordered (no cutoff)
   - Replay writes and edits, producing unified diffs at each mutation step using similar::TextDiff::from_lines()
   - Accumulate diff strings, return joined result
   - Use similar::TextDiff::from_lines(&old, &new).unified_diff().context_radius(3).header("a/path", "b/path")

7. `list_git_operations(conn, session_id: Option<&str>, operation_type: Option<&str>, limit: usize) -> Result<Vec<GitOperation>>` [for API-23, CLI-13]
   - SELECT from git_operations with optional filters
   - ORDER BY timestamp DESC, LIMIT

8. `list_git_commits(conn, session_id: Option<&str>, limit: usize) -> Result<Vec<GitOperation>>` [for API-24, API-25]
   - SELECT from git_operations WHERE operation_type = 'commit'
   - Optional session_id filter, ORDER BY timestamp DESC, LIMIT

9. `query_session_artifacts(conn, session_id: &str) -> Result<SessionArtifacts>` [for API-26, CLI-14]
   - Returns the files, git operations, AND tool executions for a session
   - SessionArtifacts struct: { files: Vec<FileEntry>, git_operations: Vec<GitOperation>, tool_executions: Vec<ToolExecutionEntry> }
   - ToolExecutionEntry struct: { tool_use_id: String, tool_name: String, input_json: Option<String>, result_summary: Option<String>, is_error: Option<bool>, message_uuid: String, timestamp: String }
   - result_summary = first 500 chars of result_content (truncated with "..." if longer)
   - Query tool_executions joined with messages for the session

10. `query_session_timeline(conn, session_id: &str, limit: usize) -> Result<Vec<TimelineEntry>>` [for API-27]
    - UNION query: SELECT from file_operations UNION ALL SELECT from git_operations UNION ALL SELECT from tool_executions
    - file_operations mapped to entry_type="file_operation" with operation_type from operation_type column
    - git_operations mapped to entry_type="git_operation" with operation_type from operation_type column
    - tool_executions mapped to entry_type="tool_execution" with operation_type=tool_name, tool_name populated, result_summary = SUBSTR(result_content, 1, 500)
    - All three joined via UNION ALL, ORDER BY timestamp ASC, LIMIT
    - The tool_executions join uses messages table for timestamp (te.message_uuid = m.uuid)

Use the existing query.rs pattern: functions take &Connection plus filter params, return Result<Vec<T>> (PAT-022). Dynamic WHERE clauses use Vec<Box<dyn rusqlite::types::ToSql>> with params_from_iter.
