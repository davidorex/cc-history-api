<task type="auto">
  <name>Task 1: Create files.rs, git.rs, artifacts.rs API handler modules</name>
  <files>crates/server/src/api/files.rs, crates/server/src/api/git.rs, crates/server/src/api/artifacts.rs</files>
  <action>
Follow the existing handler patterns in sessions.rs and search.rs. All handlers take `State(state): State<SharedState>` and use `state.conn.call(move |conn| { ... }).await` for database access. Return `Result<Json<T>, ApiError>`.

**files.rs** — 6 handlers:

1. `pub async fn list_files(State, Query<params>)` [API-17]
   - Query params: session_id, path (substring match), limit (default 100)
   - Calls artifact_queries::list_files(conn, session_id, path, limit)
   - Returns Json<Vec<FileEntry>>

2. `pub async fn file_detail(State, Path(file_id))` [API-18]
   - Calls artifact_queries::get_file(conn, file_id) for the file entry
   - Calls artifact_queries::query_file_operations(conn, file.file_path, Some(file.session_id), 1000) for ops
   - Returns Json containing file entry + operations array

3. `pub async fn file_content(State, Path(file_id), Query<params>)` [API-19]
   - Query params: at (optional message UUID), session_id (optional, needed if file_id alone is ambiguous)
   - Look up file by id to get file_path and session_id
   - Calls artifact_queries::reconstruct_file_content(conn, file_path, session_id, at)
   - Returns plain text response (Content-Type: text/plain) or 404 if no content reconstructable

4. `pub async fn file_diff(State, Path(file_id))` [API-20]
   - Look up file by id
   - Calls artifact_queries::generate_file_diff(conn, file_path, session_id)
   - Returns plain text unified diff (Content-Type: text/plain)

5. `pub async fn search_files(State, Query<params>)` [API-21]
   - Query params: q (required), limit (default 20), offset (default 0)
   - Validate q is non-empty (return 400 BadRequest if empty)
   - Calls fts::search_file_operations(conn, q, limit, offset)
   - Returns Json<Vec<FileOperationSearchResult>>

6. `pub async fn query_files(State, Json<body>)` [API-22]
   - POST body: { pattern (glob), session_id, operation_type, limit }
   - Calls artifact_queries::list_files to get candidates
   - Filter in Rust using glob::Pattern::matches_with() for the glob pattern
   - Returns Json<Vec<FileEntry>>

**git.rs** — 3 handlers:

1. `pub async fn list_git(State, Query<params>)` [API-23]
   - Query params: session_id, operation_type, limit (default 100)
   - Calls artifact_queries::list_git_operations(conn, session_id, operation_type, limit)
   - Returns Json<Vec<GitOperation>>

2. `pub async fn git_commits(State, Query<params>)` [API-24]
   - Query params: limit (default 100)
   - Calls artifact_queries::list_git_commits(conn, None, limit)
   - Returns Json<Vec<GitOperation>>

3. `pub async fn session_git_commits(State, Path(session_id))` [API-25]
   - Calls artifact_queries::list_git_commits(conn, Some(session_id), 1000)
   - Returns Json<Vec<GitOperation>>

**artifacts.rs** — 2 handlers:

1. `pub async fn session_artifacts(State, Path(session_id))` [API-26]
   - Calls artifact_queries::query_session_artifacts(conn, session_id)
   - Returns Json with files and git_operations arrays

2. `pub async fn session_timeline(State, Path(session_id), Query<params>)` [API-27]
   - Query params: limit (default 500)
   - Calls artifact_queries::query_session_timeline(conn, session_id, limit)
   - Returns Json<Vec<TimelineEntry>>

All handlers use the tokio_rusqlite::rusqlite re-export for Connection type access (per decision [03-01]). Map rusqlite errors to ApiError using the existing From implementation.

Define query parameter structs with serde::Deserialize for each handler that needs them.

Note: There is already a `mod.rs` pattern with `pub mod analytics;` etc. Follow the same naming.

Register routes in build_router. Add after the existing Schema routes, before the Events route:

```rust
// Files [API-17 through API-22]
.route("/v1/files", get(files::list_files))
.route("/v1/files/search", get(files::search_files))
.route("/v1/files/query", post(files::query_files))
.route("/v1/files/{file_id}", get(files::file_detail))
.route("/v1/files/{file_id}/content", get(files::file_content))
.route("/v1/files/{file_id}/diff", get(files::file_diff))
// Git [API-23 through API-25]
.route("/v1/git", get(git::list_git))
.route("/v1/git/commits", get(git::git_commits))
.route("/v1/git/commits/{session_id}", get(git::session_git_commits))
// Artifacts [API-26 and API-27]
.route("/v1/artifacts/{session_id}", get(artifacts_api::session_artifacts))
.route("/v1/artifacts/{session_id}/timeline", get(artifacts_api::session_timeline))
```

IMPORTANT: Route ordering matters in axum -- /v1/files/search and /v1/files/query MUST be registered BEFORE /v1/files/{file_id} to avoid the path parameter capturing "search" and "query" as file_id values. Same for /v1/git/commits before /v1/git/commits/{session_id}.

Update the module doc comment to reflect the new endpoint count (28 endpoints across 11 groups).
