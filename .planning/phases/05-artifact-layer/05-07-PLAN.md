```rust
/// List files touched by Claude Code across sessions
Files {
    /// Filter by session ID
    #[arg(long)]
    session_id: Option<String>,
    /// Filter by path substring
    #[arg(long)]
    path: Option<String>,
    /// Maximum results
    #[arg(long, default_value = "100")]
    limit: usize,
    /// Output as JSON
    #[arg(long)]
    json: bool,
},
/// Show chronological operations on a file
FileHistory {
    /// File path to show history for
    path: String,
    /// Filter by session ID
    #[arg(long)]
    session_id: Option<String>,
    /// Maximum operations to show
    #[arg(long, default_value = "50")]
    limit: usize,
    /// Output as JSON
    #[arg(long)]
    json: bool,
},
/// Reconstruct file content at a point in time
Reconstruct {
    /// File path to reconstruct
    path: String,
    /// Session ID (required -- reconstruction is per-session)
    #[arg(long)]
    session_id: String,
    /// Stop reconstruction at this message UUID
    #[arg(long)]
    at: Option<String>,
},
/// Show git operations extracted from Bash tool calls
GitLog {
    /// Filter by session ID
    #[arg(long)]
    session_id: Option<String>,
    /// Filter by git operation type (commit, push, checkout, etc.)
    #[arg(long = "type")]
    operation_type: Option<String>,
    /// Maximum operations to show
    #[arg(long, default_value = "50")]
    limit: usize,
    /// Output as JSON
    #[arg(long)]
    json: bool,
},
/// Show combined file and git artifacts for a session
Artifacts {
    /// Session ID to show artifacts for
    session_id: String,
    /// Output as JSON
    #[arg(long)]
    json: bool,
},
```

Add these to the read_cmd match block (they are all read-only and use ConnectionMode).

Create handler functions following existing patterns (run_search, run_sessions as templates):

**run_files:** Route through daemon (GET /v1/files with query params) or direct DB (artifact_queries::list_files). Use output::print_files_table or print_json.

**run_file_history:** Route through daemon or direct DB (artifact_queries::query_file_operations). Use output::print_file_operations or print_json.

**run_reconstruct:** Direct DB only (no daemon route needed for content reconstruction since it returns plain text). Call artifact_queries::reconstruct_file_content. Print content to stdout, or eprintln "No content" if None.

Note: For daemon routing of run_reconstruct, construct the URL as GET /v1/files/{file_id}/content?at={uuid}. This requires looking up the file_id first via GET /v1/files?path={path}&session_id={sid}. For simplicity in v1, reconstruct can use Direct mode only -- if the daemon is running, it still opens the DB directly for this subcommand. Or, use the daemon by first searching for the file, then requesting content. Implementer discretion on the approach.

**run_git_log:** Route through daemon (GET /v1/git with query params) or direct DB (artifact_queries::list_git_operations). Use output::print_git_operations or print_json.

**run_artifacts:** Route through daemon (GET /v1/artifacts/{session_id}) or direct DB (artifact_queries::query_session_artifacts). Use output::print_artifacts or print_json.

Update the module doc comment at the top of main.rs to list the 5 new subcommands in the usage block.

**print_files_table(files: &[FileEntry]):**
```
FILE_PATH                          SESSION         OPS  LAST_MODIFIED
-----------                        ---------       ---  -------------
/Users/david/Projects/foo/main.rs  abc123...       12   2026-02-20T10:...
```
Columns: FILE_PATH (truncated to 40 chars), SESSION (first 12 chars), OPS (operation_count), LAST_MODIFIED (24 chars).

```
TIMESTAMP                 TYPE    FILE_PATH
2026-02-20T10:15:30Z     write   /Users/.../main.rs
2026-02-20T10:16:45Z     edit    /Users/.../main.rs
```
For each op, show timestamp, operation_type, file_path. Show a content preview (first 80 chars) for write/edit ops.

**print_git_operations(ops: &[GitOperation]):**
```
TIMESTAMP                 TYPE      BRANCH     MESSAGE
2026-02-20T10:15:30Z     commit    main       feat: add authentication
2026-02-20T10:16:45Z     push      main       -
```
Columns: TIMESTAMP, TYPE (operation_type), BRANCH (or "-"), MESSAGE (commit_message truncated to 60 chars, or "-").

**print_artifacts(files: &[FileEntry], git_ops: &[GitOperation]):**
Print "Files:" header then print_files_table, then "Git Operations:" header then print_git_operations.

All formatters print to stdout. All use the same column-alignment approach as existing formatters (formatted width specifiers in println!).
