<task type="auto">
  <name>Task 2: Integrate artifact event emission and FTS file_operations rebuild into watcher_loop</name>
  <files>crates/server/src/watcher.rs</files>
  <action>
The watcher_loop currently processes JSONL file changes by calling sync_file and emitting RecordAdded/SessionStarted/SchemaDrift/VersionChanged SSE events. Extend it to also emit file/git artifact events.

**Approach:** After sync_file succeeds and records are ingested, the artifact decomposer has already run (it was wired into decompose_record in Plan 05-02). The challenge is knowing WHAT artifacts were produced so we can emit the right SSE events.

Two approaches:
1. **Query-based:** After sync_file, query the newly-created file_operations and git_operations rows for the session (those with timestamps after the sync started) and emit events for each.
2. **Return-value-based:** Modify the artifact decomposer to return a list of events it produced.

Recommend approach 1 (query-based) because it does not require modifying the store crate's decompose pipeline to return event data. The watcher already has the session_id and can query for recent operations.

Implementation:
1. Before calling sync_file, record the current timestamp (or the file's last-known sync offset)
2. After sync_file succeeds and returns records_synced > 0:
   - Query file_operations WHERE session_id = ? AND timestamp > ? for new write/edit ops
   - Query git_operations WHERE session_id = ? AND operation_type = 'commit' AND timestamp > ? for new commits
   - For each write operation: send SseEvent::FileWritten via event_tx
   - For each edit operation: send SseEvent::FileEdited via event_tx
   - For each commit: send SseEvent::GitCommit via event_tx
3. The queries run inside conn.call closures, using the same connection pool

**FTS rebuild integration:**
In the periodic FTS rebuild timer block (currently calls rebuild_fts_index every 30 seconds when new data was ingested):
- Add a call to rebuild_fts_file_operations alongside the existing rebuild_fts_index call
- Both rebuilds should run in the same conn.call closure:
```rust
conn.call(|conn| {
    claude_history_store::fts::rebuild_fts_index(conn)?;
    claude_history_store::fts::rebuild_fts_file_operations(conn)?;
    Ok(())
}).await
```

Update the module doc comment to mention the 3 new SSE event types and the file_operations FTS rebuild.
  </action>
