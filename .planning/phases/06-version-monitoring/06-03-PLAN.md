**In `check_version_change()`**, after the SSE event emission (after the `event_tx.send(SseEvent::VersionChanged {...})` call), add version_history persistence:

```rust
// Persist version change to version_history table
let ver = new_version.clone();
let sid = session_id.to_string();
let persist_result = conn.call(move |conn| {
    conn.execute(
        "INSERT INTO version_history (version, first_seen_at, last_seen_at, session_id, session_count)
         VALUES (?1, datetime('now'), datetime('now'), ?2, 1)
         ON CONFLICT(version) DO UPDATE SET
           last_seen_at = datetime('now'),
           session_count = version_history.session_count + 1",
        rusqlite::params![ver, sid],
    )
}).await;

if let Err(e) = persist_result {
    tracing::warn!(
        error = %e,
        version = %new_version,
        "Failed to persist version change to version_history table — SSE event was still emitted"
    );
}
```

Key design points:
- Persistence happens AFTER SSE event emission so a DB failure doesn't prevent event delivery
- INSERT ON CONFLICT handles both first-seen and re-observed versions
- session_count increments on each observation (may count same session multiple times during rapid ingestion — acceptable for a count metric)
- Failure is logged at warn level but does NOT abort the version check flow

**In `watcher_loop()`**, after initializing the WatcherState (around line 473, after the initial_version block), add a startup backfill of version_history:

```rust
// Backfill version_history from sessions table on startup.
// This ensures version_history is populated even if the migration's
// backfill ran on an empty database (first-time setup).
let backfill_result = conn.call(|conn| {
    conn.execute_batch(
        "INSERT OR IGNORE INTO version_history (version, first_seen_at, last_seen_at, session_id, session_count)
         SELECT
             version,
             MIN(first_seen_at),
             MAX(COALESCE(last_seen_at, first_seen_at)),
             (SELECT s2.session_id FROM sessions s2 WHERE s2.version = sessions.version
              ORDER BY s2.first_seen_at ASC LIMIT 1),
             COUNT(*)
         FROM sessions
         WHERE version IS NOT NULL AND version != ''
         GROUP BY version"
    )
}).await;

match backfill_result {
    Ok(()) => tracing::info!("version_history backfill completed on startup"),
    Err(e) => tracing::warn!(error = %e, "version_history backfill failed on startup — will be populated incrementally"),
}
```

This backfill uses INSERT OR IGNORE so it's safe to run repeatedly — only populates rows that don't already exist.

Update the module-level doc comment to mention version_history persistence alongside SSE event emission.

```rust
/// Enhanced version history entry from the version_history table.
#[derive(Debug, Serialize, Deserialize)]
pub struct VersionHistoryEntry {
    pub version: String,
    pub first_seen_at: String,
    pub last_seen_at: String,
    pub session_id: Option<String>,
    pub session_count: i64,
    pub new_fields_count: i64,
}

/// A drift field with occurrence count and promotion status.
#[derive(Debug, Serialize, Deserialize)]
pub struct DriftFieldEntry {
    pub field_name: String,
    pub record_type: String,
    pub sample_value: Option<String>,
    pub occurrence_count: i64,
    pub first_seen_at: String,
    pub promotion_status: String,  // "promoted", "extra_json", or "unhandled"
}

/// Drift entries grouped by version, then by record type within each version.
#[derive(Debug, Serialize, Deserialize)]
pub struct VersionDriftGroup {
    pub version: String,
    pub record_types: Vec<RecordTypeDriftGroup>,
}

/// Drift entries for a specific record type within a version.
#[derive(Debug, Serialize, Deserialize)]
pub struct RecordTypeDriftGroup {
    pub record_type: String,
    pub fields: Vec<DriftFieldEntry>,
}

/// Version diff entry showing fields introduced or disappeared in a version.
#[derive(Debug, Serialize, Deserialize)]
pub struct VersionDiffEntry {
    pub version: String,
    pub first_seen_at: String,
    pub last_seen_at: String,
    pub session_id: Option<String>,
    pub session_count: i64,
    pub new_fields_count: i64,
    pub new_fields: Vec<String>,
    pub disappeared_fields: Vec<String>,
}
```

**Add `version_history_enhanced()` function:**
Query the `version_history` table ORDER BY first_seen_at ASC. Returns `Vec<VersionHistoryEntry>`. This replaces the old `version_history()` function's role for Phase 6 consumers but the old function is kept for backward compatibility.

**Add `version_history_with_diff()` function:**
For each version in version_history, query schema_drift_log to find:
- `new_fields`: fields with this version that don't exist in any earlier version
- `disappeared_fields`: fields from the previous version that don't appear in this version
Returns `Vec<VersionDiffEntry>`.

**Add `drift_by_version()` function:**
1. Query schema_drift_log grouped by version, record_type, with occurrence_count
2. For each field, compute promotion_status dynamically:
   - Check if field_name matches a column on the target table (messages, token_usage, etc.) using PRAGMA table_info. Map record_type to table name: "user" -> "messages", "assistant" -> "messages", "assistant.message" -> "messages", "assistant.message.usage" -> "token_usage", etc.
   - If field matches a real column: "promoted"
   - If the target table has an extra_json column: "extra_json"
   - Otherwise: "unhandled"
3. Group results in Rust by version -> record_type -> Vec<DriftFieldEntry>
4. Truncate sample_value to ~200 chars per CONTEXT decision

Returns `Vec<VersionDriftGroup>`.

The promotion status computation uses a helper function that builds a HashSet of column names from PRAGMA table_info for relevant tables. This is computed once per query invocation (not per field).
